## プロダクトの一言説明

> **「なんて言えばいいかわからない要望」を、AIが自動でシフト計算のルールに翻訳するエージェント**

---

## 解決したい課題

飲食店のシフト管理では、スタッフからこういう声が日常的に出る：

- 「毎週水曜は習い事があって入れません」
- 「試験期間中はなるべく入れないでほしい」
- 「22時以降のシフトは月3回までにして」

これらを既存のシフト管理ツールで表現しようとすると、**1日ずつ手動で設定する**しかない。
手間がかかる上に、新しいパターンが来るたびに管理者が手作業で対応しなければならない。

---

## このプロダクトの仕組み（3ステップ）

```
① スタッフが自然言語で入力
   「毎週水曜は入れません」

② AIが翻訳 + 未知ルールの場合は自動でコードを書く
   → 既知のルール: すぐ計算に使える形に変換
   → 未知のルール: AIがその場で処理ロジックを自動生成 → 人間が承認

③ OR-Toolsが最適なシフトを計算
   → 全員の希望を満たしつつ、必要人数を確保したシフトを出力
```

---

## ハッカソンのコアポイント（審査員向け）

**「AIが自分でコードを書いて、自分でテストして、人間に承認を求める」**

通常のAIツールは「翻訳するだけ」。
このエージェントは**新しいルールに直面したとき、自分でその処理ロジック（ハンドラ）を書き、テストし、動いたら人間に承認を求める**。
承認後は次回から自動で使い回す。これが自律エージェントの証明。

## 画面構成（マルチページ・ハブ `src/ui/app.py` で1つのURLに統合）

`python -m streamlit run src/ui/app.py` で起動し、サイドバーから切り替える。

```
① セットアップ画面        ✅ マスタCSVアップロード + 営業情報フォーム
② シフト作成の要望        ✅ 全体方針(新人はキッチン優先等)+横断ルール+特殊事情を自然言語で → 反映済み/確認中
③ 出勤希望CSV登録         ✅ desired_shifts.csv→availability保存（note列は保存・AI解釈は次）
④ 管理者承認画面          ✅ 未知→AIがハンドラ生成→テスト→承認/却下（デモのクライマックス）
⑤ シフト確認画面          ✅ 保存済みデータから計算→シフト表（暫定/確定バッジ）
```

> **役割整理（2026-06-04）**: ②は「CSVの列に収まらない例外・人間関係・繰り返し・未知ルール」
> （経路C）の入力に専念。日ごとの希望（経路B）はCSV、シフト表の表示は⑤、と画面の責務を分離した。
