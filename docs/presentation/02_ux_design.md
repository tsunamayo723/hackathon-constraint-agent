# 発表用サマリ 02 — UX設計：「AIは分かったフリをしない」

*最終更新: 2026-05-28*

---

## ハッカソンの差別化ポイント

> 普通のAIサービスは「分からない部分を黙って無視する」。
> このエージェントは**「分からなかった部分を、元の言葉のままユーザーに見せる」**。

---

## 解決した課題

### よくあるAIサービスの問題

スタッフが「毎週水曜は習い事で休みたい」と入力 →
AIが部分的にしか理解できなかったとき、よくある実装は：

❌ **Bad UX 1**: 黙って無視してシフトを作る → 水曜に入れられる → 信頼崩壊
❌ **Bad UX 2**: 「処理できません」と全部を弾く → 店舗が回らない
❌ **Bad UX 3**: 管理者の承認を待つまでシフト作成不可 → 数日待ち

### このエージェントの解決法

✅ **暫定シフト + 透明性**: 翻訳できた部分でシフトを作り、未翻訳の部分は「保留中」として元の言葉のまま表示。サイト管理者が承認したら自動で再計算される。

---

## ユーザー画面のイメージ

```
あなたが入力した要望:
「ランチに4人入れて。毎週水曜は習い事で休みです。」

✅ 反映済み
  └ 「ランチに4人入れて」
     → ホール 11:00〜14:00 / 4名 で計算しました

⏳ 確認中
  └ 「毎週水曜は習い事で休みです」
     → サイト管理者が新ルールを準備中です
     → 確定次第、自動でシフトに反映してお知らせします

【暫定シフト】2026年11月分
（このシフトは確認中の要望が反映されていない暫定版です）
```

---

## ハッカソン審査軸への対応

| 審査軸 | このUX設計の意味 |
|---|---|
| **AIの自律性** | 「分からない」を自分で検出 → 自分でハンドラを書く → 承認を求める |
| **実用性** | 待たせない・騙さない。実サービスにそのまま乗せられる設計 |
| **AIの必然性** | 自然言語の解釈・コード生成・自信度判定はAIでないと不可能 |
| **DevOps的「まわす」** | 未知タイプ承認 → 再計算 → ユーザー通知が自動ループ |

## 実装の構成要素

```
┌─────────────────┐
│ パーサ           │ → translated / untranslated に振り分け
│ (Gemini Flash)   │
└────────┬─────────┘
         ↓
┌─────────────────┐    ┌────────────────────┐
│ ソルバー         │    │ サイト管理者キュー │
│ (OR-Tools)       │    │ (PendingTypeReq)   │
│ → 暫定シフト     │    │ → 承認待ち一覧      │
└─────────────────┘    └──────────┬─────────┘
                                  ↓ 承認
                       ┌────────────────────┐
                       │ AIエージェント      │
                       │ (Gemini Pro)        │
                       │ ハンドラ自動生成    │
                       └──────────┬─────────┘
                                  ↓
                       ┌────────────────────┐
                       │ 自動再計算          │
                       │ → 確定シフトへ昇格  │
                       └────────────────────┘
```

## デモシナリオ（7分）

1. スタッフ画面で「毎週水曜は習い事…」と入力
2. パーサが既知/未知に振り分けるのを画面で見せる
3. 暫定シフトが「保留中」付きで表示される
4. **画面を切り替え**: サイト管理者画面に承認待ちが表示
5. AIが生成したハンドラコード・テスト結果・自信度を見せる
6. 「承認」ボタンを押す
7. 自動再計算され、暫定 → 確定 になり、保留項目がシフトに反映される
