## 🎯 採用テーマ（決定）

### 「制約管理エージェント」

自然言語 → JSON構造化 → 自動スキーマ進化 → ハンドラ自動生成 → 検証 → 人間承認 までを一貫して行う独立プロダクト

既存のシフト管理ツールから着想を得て、より汎用的な「定性情報を構造化して処理可能にする」エージェントとして開発する。

→ 詳細な設計は [02_system.md](./02_system.md) 参照

## 💡 アイデア一覧

### Idea 1｜定性情報→JSON→プログラム変換 ⭐採用

- **概要**: 自然言語をJSON形式に構造化し、AIが苦手な領域の問題を解くプログラムに渡す
- **ヒント元**: シフト管理における自然言語→JSON変換
- **発展**: 「制約タイプ + パラメータ」構造で柔軟性を確保し、未知の型はAIが自律的に拡張
- **実現性**: ★★★★☆
- **ハッカソン要件マッチ**: ★★★★☆（DevOps文脈にCI/CD的サイクルとして組み込み可能）
- **面白さ**: ★★★★★

### Idea 2｜自然言語→SQL生成（保留）

- 営業の要件定義書をJSON化→過去SQLを学習させて自動生成
- 本職（vpoint）の業務課題ベース
- **保留理由**: 「自然言語→SQL」は競合多数。独自性確保が課題

### Idea 3｜自動モデル更新エージェント（保留）

- 機械学習モデルの実績データから自動で改訂提案
- DevOps直球
- **保留理由**: データソース確保が最大の壁

## 📌 採用までの経緯

1. 当初3案を比較、Idea 1の独創性とIdea 2の業務インパクトを掛け合わせる方向で検討
